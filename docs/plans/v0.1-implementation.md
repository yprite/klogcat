# klogcat v0.1 Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Build klogcat v0.1: a Tauri + React + TypeScript desktop app that tails a single Kubernetes pod internal APP/ACC/ERR log file via `kubectl exec ... tail -n <N> -F <file>`, parses JSON Lines best-effort, and provides raw-line grep/filtering.

**Architecture:** Rust/Tauri owns `kubectl` discovery, settings persistence, child process lifecycle, stdout/stderr byte streaming, and line splitting. React/TypeScript owns UI state, settings form state, parsing/rendering JSON log rows, ring buffer, grep, pause/resume, and stale event discard using `streamId`. v0.1 supports exactly one active stream at a time.

**Tech Stack:** Tauri 2, Rust, React, TypeScript, Vite, Zustand, Tailwind CSS, `@tanstack/react-virtual`, Vitest, React Testing Library.

**Source of truth:** `docs/DESIGN.md`.

## 0. Non-negotiable v0.1 decisions

Implementation must preserve these decisions from `docs/DESIGN.md`:

```text
- This is NOT a kubectl logs viewer.
- The only log collection command is:
  kubectl exec -n <namespace> <pod> -c <container> -- tail -n <N> -F <filePath>
- v0.1 has one selected namespace, one selected pod, one active source type, and at most one active backend stream.
- Source type values are exactly: app, access, error.
- UI labels are exactly: APP, ACC, ERR.
- Source type is separate from JSON internal logType.
- v0.1 grep is raw-line only: query.trim().toLowerCase() against raw.toLowerCase().
- No regex, no case-sensitive toggle, no invert match, no structured query language.
- stdout line splitting is byte-buffer based, not chunk-string based.
- stderr lines are diagnostics only; stderr alone is not fatal.
- start validation/spawn failures reject the command and do not emit duplicate log://error.
- spawn-after runtime failures use log://exit or log://error events.
- requestedStop=true exits are never fatal.
```

Implementation-plan decisions that close remaining P1 ambiguity:

```text
- Running 중 Start 버튼은 disabled 처리한다. 새 Start는 Stop 완료 후에만 가능하다.
- Source/namespace/pod 변경 시 frontend awaits stopLogStream(activeStreamId), clears buffer, then sets streamStatus = stopped.
- Backend emits log://started exactly once after spawn success and active stream registration.
- startLogStream resolves after spawn success and registration are complete. Frontend running transition is idempotent across resolve/log://started.
- Frontend stores ActiveStreamMeta keyed by activeStreamId. LogLineEvent remains minimal and parser combines event + ActiveStreamMeta.
- Common command reject shape is CommandError.
- Test name is "partial line without trailing newline is flushed on process exit".
```

```text
- After settings and namespaces load, if settings.defaultNamespace exists and is present in namespaces, select it.

- If settings.defaultNamespace is absent, leave namespace unselected; do not silently pick the first namespace.
```

**Step 4: Implement validation**

`src/config/validateSettings.ts`:

The frontend validator must match backend strict-schema behavior so the UI never accepts settings that `saveSettings` will reject.

```ts
import type { PersistedSettings, SettingsValidationError } from '../types/settings'

const sourceKeys = ['app', 'access', 'error'] as const

function isRecord(value: unknown): value is Record<string, unknown> {
  return typeof value === 'object' && value !== null && !Array.isArray(value)
}

function rejectExtraKeys(value: Record<string, unknown>, allowed: readonly string[], prefix: string, errors: SettingsValidationError[]) {
  for (const key of Object.keys(value)) {
    if (!allowed.includes(key)) errors.push({ field: `${prefix}.${key}`, message: `Unknown key: ${key}` })
  }
}

export function validateSettings(value: unknown): SettingsValidationError[] {
  const errors: SettingsValidationError[] = []

  if (!isRecord(value)) return [{ field: 'settings', message: 'Settings must be an object' }]

  rejectExtraKeys(value, ['schemaVersion', 'defaultNamespace', 'initialTailLines', 'bufferLimit', 'logSources'], 'settings', errors)

  if (value.schemaVersion !== 1) errors.push({ field: 'schemaVersion', message: 'schemaVersion must be 1' })

  if (!Number.isInteger(value.initialTailLines) || (value.initialTailLines as number) < 0 || (value.initialTailLines as number) > 100000) {
    errors.push({ field: 'initialTailLines', message: 'initialTailLines must be 0..100000' })
  }

  if (!Number.isInteger(value.bufferLimit) || (value.bufferLimit as number) < 1000 || (value.bufferLimit as number) > 200000) {
    errors.push({ field: 'bufferLimit', message: 'bufferLimit must be 1000..200000' })
  }

  if (value.defaultNamespace !== undefined && typeof value.defaultNamespace !== 'string') {
    errors.push({ field: 'defaultNamespace', message: 'defaultNamespace must be a string when provided' })
  }

  const logSources = value.logSources
  if (!isRecord(logSources)) {
    errors.push({ field: 'logSources', message: 'logSources must be an object' })
    return errors
  }

  const actualKeys = Object.keys(logSources).sort()
  const expectedKeys = [...sourceKeys].sort()
  if (actualKeys.join(',') !== expectedKeys.join(',')) {
    errors.push({ field: 'logSources', message: 'logSources must contain exactly app/access/error keys' })
  }

  for (const key of sourceKeys) {
    const source = logSources[key]
    if (!isRecord(source)) {
      errors.push({ field: `logSources.${key}`, message: 'source config must be an object' })
      continue
    }
    rejectExtraKeys(source, ['container', 'filePath'], `logSources.${key}`, errors)
    if (typeof source.container !== 'string' || source.container.trim() === '') {
      errors.push({ field: `logSources.${key}.container`, message: 'container is required' })
    }
    if (typeof source.filePath !== 'string' || !source.filePath.startsWith('/') || source.filePath.includes('\0')) {
      errors.push({ field: `logSources.${key}.filePath`, message: 'filePath must be an absolute path without null bytes' })
    }
  }

  return errors
}

export function assertValidSettings(value: unknown): asserts value is PersistedSettings {
  const errors = validateSettings(value)
  if (errors.length > 0) throw Object.assign(new Error('settings validation failed'), { validationErrors: errors })
}
```

```ts
export function normalizeGrepQuery(query: string): string {
  return query.trim().toLowerCase()
}

export function matchesGrep(raw: string, query: string): boolean {
  const normalized = normalizeGrepQuery(query)
  if (normalized === '') return true
  return raw.toLowerCase().includes(normalized)
}
```

```ts
import { parseAppLog } from './parseAppLog'
import { parseErrorLog } from './parseErrorLog'

export type ParsedLogLineWithoutId = Omit<ParsedLogLine, 'id'>

export function parseLogLine(
  raw: string,
  sourceType: SourceLogType,
  sourceMeta: SourceMeta,
  receivedAt: number,
): ParsedLogLineWithoutId {
  try {
    const json = JSON.parse(raw)

  } catch {

      ...sourceMeta,
      raw,
      parseStatus: 'raw',
      receivedAt,
      summary: raw,
    }
  }
}
```

```ts
export type RingBufferResult<T> = { items: T[]; dropped: number }

export function appendWithLimit<T>(items: T[], item: T, limit: number): RingBufferResult<T> {
  const next = [...items, item]
  if (next.length <= limit) return { items: next, dropped: 0 }
  const dropped = next.length - limit

}
```

### Task 16: Wire Tauri stream events into log store

**Objective:** Subscribe to Tauri log events and update the log store with stale event protection.

**Files:**
- Create: `src/commands/tauriLogEvents.ts`
- Modify: `src/App.tsx`
- Modify: `src/stores/logStore.ts`

**Step 1: Add event subscription helper**

Use `@tauri-apps/api/event`:

```ts
import { listen } from '@tauri-apps/api/event'
import type { LogLineEvent, LogStreamExitEvent, LogStreamStartedEvent, LogStreamStderrEvent, LogStreamErrorEvent } from '../types/log'

export async function subscribeLogEvents(handlers: {
  onStarted: (event: LogStreamStartedEvent) => void
  onLine: (event: LogLineEvent) => void
  onStderr: (event: LogStreamStderrEvent) => void
  onExit: (event: LogStreamExitEvent) => void
  onError: (event: LogStreamErrorEvent) => void
}) {
  const unlisteners = await Promise.all([
    listen<LogStreamStartedEvent>('log://started', (e) => handlers.onStarted(e.payload)),
    listen<LogLineEvent>('log://line', (e) => handlers.onLine(e.payload)),
    listen<LogStreamStderrEvent>('log://stderr', (e) => handlers.onStderr(e.payload)),
    listen<LogStreamExitEvent>('log://exit', (e) => handlers.onExit(e.payload)),
    listen<LogStreamErrorEvent>('log://error', (e) => handlers.onError(e.payload)),
  ])
  return () => unlisteners.forEach((unlisten) => unlisten())
}
```

**Step 2: Apply stale event rule**

All handlers must ignore `event.streamId !== activeStreamId` except global errors without a streamId.

```ts
const source = settings.logSources[selectedSourceType]
const selectedPodInfo = pods.find((p) => p.name === pod)

  showStartError('pod_not_found')
  return
}

if (selectedPodInfo.phase !== 'Running') {

  return
}

  showStartError('container_not_found')
  return

logStore.prepareStarting(meta)
```
